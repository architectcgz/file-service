# 文件上传服务边界测试 - 快速开始指南

本指南帮助你快速开始运行文件上传服务的边界场景测试。

---

## 1. 前置条件检查

### 启动基础设施

```powershell
cd file-service/docker
docker-compose up -d
```

### 验证服务

```powershell
# PostgreSQL
docker exec -it file-service-postgres pg_isready -U postgres

# Redis  
docker exec -it file-service-redis redis-cli -a redis123456 ping

# RustFS
curl http://localhost:9001/minio/health/live
```

---

## 2. 启动File Service

```powershell
cd file-service/file-service
mvn spring-boot:run
```

验证:
```powershell
curl http://localhost:8089/actuator/health
```

---

## 3. 运行测试

### 单个测试

```powershell
cd file-service/file-service/test

# 断点续传测试
.\测试-断点续传场景.ps1

# 网络异常测试
.\测试-网络异常场景.ps1

# 资源限制测试
.\测试-资源限制场景.ps1
```

### 所有测试

```powershell
# 运行所有测试并生成报告
.\运行所有边界测试.ps1 -GenerateReport
```

---

## 4. 查看结果

测试报告位置:
```
file-service/file-service/test/reports/boundary_tests_report_YYYYMMDDHHMMSS.md
```

---

## 5. 故障排查

### 服务连接失败
```powershell
curl http://localhost:8089/actuator/health
```

### 数据库问题
```powershell
docker-compose restart postgres
```

### Redis问题
```powershell
docker-compose restart redis
```

### S3问题
```powershell
docker-compose restart rustfs
```
